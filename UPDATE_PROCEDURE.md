# Процедура обновления проекта коррекции чеков ККТ

## Обзор

Данный документ описывает процедуру безопасного обновления проекта, включая резервное копирование, тестирование и проверку работоспособности.

## Подготовка к обновлению

### 1. Резервное копирование

#### Создание резервной копии исходного кода
```bash
# Создание архива с текущей версией
zip -r backup_source_$(date +%Y%m%d_%H%M%S).zip *.py *.md *.txt *.xlsx *.csv *.json *.log docs/
```

#### Создание резервной копии файлов данных
```bash
# Копирование всех важных файлов данных
mkdir backup_data_$(date +%Y%m%d_%H%M%S)
copy list.csv backup_data_$(date +%Y%m%d_%H%M%S)/
copy receipts_data.csv backup_data_$(date +%Y%m%d_%H%M%S)/
copy processed.json backup_data_$(date +%Y%m%d_%H%M%S)/
copy correction_process.log backup_data_$(date +%Y%m%d_%H%M%S)/
copy vat10_fp.txt backup_data_$(date +%Y%m%d_%H%M%S)/
```

#### Резервное копирование конфигурационных файлов
- Создайте резервные копии всех файлов с настройками
- Особое внимание уделите файлам `correction_final.py`, `prepare_data.py`, `parse_ofd_receipts.py`, где могут быть измененные настройки

### 2. Проверка текущей версии

#### Проверка версии Python
```bash
python --version
```

#### Проверка установленных пакетов
```bash
pip list | findstr -i "pandas openpyxl requests pywin32"
```

#### Проверка работоспособности текущей версии
```bash
# Проверка синтаксиса всех Python-файлов
python -m py_compile correction_final.py
python -m py_compile prepare_data.py
python -m py_compile parse_ofd_receipts.py
```

## Процесс обновления

### 1. Обновление исходного кода

#### Если используется Git
```bash
# Сохранение локальных изменений
git stash

# Получение последней версии
git pull origin main

# Восстановление локальных изменений
git stash pop
```

#### Если обновление вручную
- Замените файлы в соответствии с новой версией
- Убедитесь, что не потеряли локальные изменения в настройках

### 2. Обновление зависимостей

#### Установка новых или обновленных зависимостей
```bash
pip install -r requirements.txt
# или
pip install pandas openpyxl requests pywin32
```

#### Проверка совместимости версий
- Убедитесь, что новые версии пакетов совместимы с кодом
- Проверьте, не вносят ли изменения в поведение зависимостей

### 3. Обновление конфигурационных файлов

#### Проверка изменений в настройках
- Сравните новые шаблоны конфигурационных файлов с вашими локальными настройками
- Перенесите ваши локальные изменения в новые версии файлов
- Обновите настройки в соответствии с новыми требованиями

## Тестирование обновленной версии

### 1. Проверка синтаксиса и базовой функциональности

#### Проверка синтаксиса
```bash
python -m py_compile correction_final.py
python -m py_compile prepare_data.py
python -m py_compile parse_ofd_receipts.py
```

#### Проверка импорта модулей
```bash
python -c "import correction_final; print('correction_final imported successfully')"
python -c "import prepare_data; print('prepare_data imported successfully')"
python -c "import parse_ofd_receipts; print('parse_ofd_receipts imported successfully')"
```

### 2. Тестирование подготовки данных

#### Запуск теста подготовки данных
```bash
# Создание резервной копии существующего list.csv
copy list.csv list.csv.backup

# Запуск подготовки данных
python prepare_data.py

# Проверка результата
type list.csv
```

#### Проверка корректности созданного файла
- Убедитесь, что файл `list.csv` создан с правильной структурой
- Проверьте, что все фискальные признаки извлечены корректно
- Убедитесь, что типы оплаты определены правильно

### 3. Тестирование парсинга чеков (если используется)

#### Запуск теста парсинга
```bash
# Создание резервной копии существующего receipts_data.csv
copy receipts_data.csv receipts_data.csv.backup

# Запуск парсинга чеков
python parse_ofd_receipts.py

# Проверка результата
type receipts_data.csv
```

#### Проверка корректности созданного файла
- Убедитесь, что файл `receipts_data.csv` создан с правильной структурой
- Проверьте, что данные о товарах извлечены корректно
- Убедитесь, что даты и суммы соответствуют ожидаемым

### 4. Тестирование коррекции чеков (в безопасном режиме)

#### Подготовка к тестированию
- Убедитесь, что `MODE = 'test'` в файле `correction_final.py`
- Убедитесь, что в тестовом режиме будет обработан только 1 чек
- Подготовьте тестовую ККТ или используйте симулятор

#### Запуск теста коррекции
```bash
# Запуск в тестовом режиме
python correction_final.py
```

#### Проверка результата
- Проверьте файл `correction_process.log` на наличие ошибок
- Убедитесь, что чек коррекции был успешно пробит
- Проверьте, что логика коррекции работает правильно

## Проверка работоспособности

### 1. Функциональные проверки

#### Проверка всех основных функций
- Подготовка данных из Excel
- Парсинг чеков с Платформы ОФД
- Коррекция чеков (в тестовом режиме)
- Логирование операций
- Сохранение состояния обработки

#### Проверка обработки ошибок
- Попробуйте запустить скрипты с некорректными данными
- Убедитесь, что ошибки обрабатываются корректно
- Проверьте, что логирование ошибок работает правильно

### 2. Проверка производительности

#### Проверка времени выполнения
- Сравните время выполнения основных операций с предыдущей версией
- Убедитесь, что производительность не ухудшилась значительно

#### Проверка использования ресурсов
- Мониторьте использование памяти при обработке больших файлов
- Убедитесь, что нет утечек памяти

## Завершение обновления

### 1. Финальная проверка

#### Проверка полного цикла
- Протестируйте полный цикл: подготовка данных → парсинг → коррекция
- Убедитесь, что все этапы работают корректно
- Проверьте, что результаты соответствуют ожиданиям

#### Проверка восстановления после прерывания
- Прервите процесс обработки на середине
- Повторно запустите скрипт
- Убедитесь, что обработка продолжается с места прерывания

### 2. Обновление документации

#### Обновление конфигурации
- Обновите `CONFIG_SUMMARY.md` с новыми настройками
- Обновите `CONFIGURATION.md` с новыми параметрами

#### Обновление справочника API
- Обновите `API_REFERENCE.md` с новыми функциями или изменениями
- Обновите описания параметров и возвращаемых значений

#### Обновление руководства по установке
- Обновите `INSTALLATION.md` с новыми требованиями
- Добавьте информацию о новых зависимостях

### 3. Очистка

#### Удаление временных файлов
- Удалите временные файлы, созданные в процессе тестирования
- Оставьте резервные копии на случай необходимости отката

#### Обновление информации о версии
- Обновите информацию о версии в документации
- Зафиксируйте изменения в системе контроля версий (если используется)

## Возможные проблемы и решения

### Проблемы с совместимостью

#### Несовместимые изменения в API
- Проверьте, не изменились ли интерфейсы функций
- Обновите вызовы функций в соответствии с новым API

#### Изменения в формате данных
- Убедитесь, что старые файлы данных совместимы с новой версией
- При необходимости добавьте преобразование формата

### Проблемы с зависимостями

#### Конфликты версий пакетов
- Убедитесь, что все зависимости совместимы друг с другом
- При необходимости зафиксируйте точные версии в `requirements.txt`

#### Проблемы с pywin32
- Если возникли проблемы с COM-объектами, переустановите pywin32:
```bash
pip uninstall pywin32
pip install pywin32
python Scripts/pywin32_postinstall.py -install
```

### Проблемы с подключением к ККТ

#### Изменения в драйвере ККТ
- Убедитесь, что драйвер ККТ совместим с новой версией
- Проверьте настройки подключения

## Откат обновления

### Если обновление неудачное

#### Восстановление из резервной копии
```bash
# Восстановление исходного кода
unzip backup_source_YYYYMMDD_HHMMSS.zip

# Восстановление файлов данных
copy backup_data_YYYYMMDD_HHMMSS\list.csv .
copy backup_data_YYYYMMDD_HHMMSS\receipts_data.csv .
copy backup_data_YYYYMMDD_HHMMSS\processed.json .
```

#### Восстановление зависимостей
```bash
pip install -r requirements_backup.txt
```
