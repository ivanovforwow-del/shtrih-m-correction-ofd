# Карта документации проекта коррекции чеков ККТ

## Обзор

Этот документ описывает всю документацию проекта и её назначение, чтобы облегчить ориентацию в имеющихся материалах.

## Основные документы

### `README.md`
- **Назначение:** Основная документация проекта
- **Содержание:** Общее описание, назначение, структура проекта, требования и инструкции по использованию
- **Аудитория:** Все пользователи проекта

### `FILE_DESCRIPTIONS.md`
- **Назначение:** Описание всех файлов проекта
- **Содержание:** Подробное описание каждого файла в проекте и его функций
- **Аудитория:** Разработчики и администраторы

## Документация в директории `docs/`

### `docs/DETAILED_DOCS.md`
- **Назначение:** Подробная документация проекта
- **Содержание:** Технические детали, архитектурные особенности и глубокое описание всех компонентов системы
- **Аудитория:** Разработчики и системные администраторы

### `docs/API_REFERENCE.md`
- **Назначение:** Справочник API проекта
- **Содержание:** Документация всех функций, классов и методов проекта с описанием параметров и возвращаемых значений
- **Аудитория:** Разработчики

### `docs/CONFIGURATION.md`
- **Назначение:** Документация по настройке проекта
- **Содержание:** Описание всех конфигурационных параметров и настроек системы
- **Аудитория:** Администраторы и разработчики

### `docs/INSTALLATION.md`
- **Назначение:** Инструкция по установке и запуску проекта
- **Содержание:** Пошаговое руководство по настройке и запуску системы
- **Аудитория:** Администраторы и пользователи

### `docs/TROUBLESHOOTING.md`
- **Назначение:** Решение проблем и часто задаваемые вопросы
- **Содержание:** Описание типичных ошибок и способы их решения
- **Аудитория:** Все пользователи проекта

### `docs/PROJECT_STRUCTURE.md`
- **Назначение:** Документация по структуре проекта
- **Содержание:** Описание архитектуры и взаимодействия компонентов
- **Аудитория:** Разработчики и архитекторы

### `docs/DEVELOPMENT.md`
- **Назначение:** Руководство для разработчиков
- **Содержание:** Процессы разработки, тестирования и внесения изменений в проект
- **Аудитория:** Разработчики

## Обновление документации

При внесении изменений в проект, следующие документы могут потребовать обновления:

- `README.md` - при изменениях в основной функциональности
- `docs/API_REFERENCE.md` - при изменениях в интерфейсах
- `docs/CONFIGURATION.md` - при добавлении новых настроек
- `docs/TROUBLESHOOTING.md` - при появлении новых проблем
- `CSV_FORMATS.md` - при изменениях в форматах данных
- `COMMANDS.md` - при добавлении новых команд
- `CONFIG_SUMMARY.md` - при добавлении новых настроек

## Рекомендации по использованию

- Начинайте с `README.md` для общего понимания проекта
- Используйте `docs/TROUBLESHOOTING.md` при возникновении любых проблем
- Обращайтесь к `COMMANDS.md` для быстрого доступа к часто используемым командам
- Для глубокого понимания архитектуры используйте `docs/PROJECT_STRUCTURE.md` и `docs/DETAILED_DOCS.md`
