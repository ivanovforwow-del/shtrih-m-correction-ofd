# Форматы CSV-файлов проекта коррекции чеков ККТ

## Обзор

Данный документ описывает форматы всех CSV-файлов, используемых в проекте.

## list.csv

### Назначение
Основной файл с подготовленными данными чеков для коррекции. Создается скриптом `prepare_data.py` из Excel-файла.

### Формат
Файл использует разделитель точка с запятой (`;`) и кодировку UTF-8 с BOM.

### Структура
| Поле | Тип | Описание | Пример |
|------|-----|----------|--------|
| `summ` | float | Сумма чека в рублях | `1799.55` |
| `type` | int | Тип оплаты: 0 = карта (безнал), 1 = наличные | `0` |
| `fiscal_sign` | string | Фискальный признак чека (ФП) | `2444685719` |

### Пример содержимого
```csv
summ;type;fiscal_sign
0.37;0;2703755211
1799.55;0;2444685719
1000.00;1;2361559314
```

### Особенности
- Первый столбец всегда содержит сумму чека
- Второй столбец определяет тип оплаты (0 - безналичные, 1 - наличные)
- Третий столбец содержит фискальный признак чека (только цифры)
- Сумма округляется до 2 знаков после запятой

## receipts_data.csv

### Назначение
Файл с детальными данными о товарах из чеков. Создается скриптом `parse_ofd_receipts.py` путем парсинга чеков с Платформы ОФД.

### Формат
Файл использует разделитель точка с запятой (`;`) и кодировку UTF-8 с BOM.

### Структура
| Поле | Тип | Описание | Пример |
|------|-----|----------|--------|
| `row` | int | Номер строки в исходном Excel | `2` |
| `fp` | string | Фискальный признак чека | `2703755211` |
| `name` | string | Наименование товара | `АИ-92-К5` |
| `quantity` | float | Количество (в литрах или штуках) | `27.0` |
| `unit` | string | Единица измерения: `л` = литры, `шт` = штуки | `л` |
| `price` | float | Цена за единицу | `66.65` |
| `summ` | float | Сумма позиции | `1799.55` |
| `date` | string | Дата и время чека в формате `YYYY-MM-DD HH:MM:SS` | `2026-01-01 00:06:00` |

### Пример содержимого
```csv
row;fp;name;quantity;unit;price;summ;date
2;2703755211;ДТ-А-К5;0.004;л;92.5;0.37;2026-01-01 00:06:00
3;2444685719;АИ-92-К5;27.0;л;66.65;1799.55;2026-01-01 00:06:00
10;1852047038;АИ-95-К5;45.0;л;66.7;3001.5;2026-01-01 00:16:00
10;1852047038;Капучино (Время Кофе) с корицей 300мл;0;л;0;220.0;2026-01-01 00:16:00
```

### Особенности
- Один фискальный признак (`fp`) может встречаться в нескольких строках (много товаров в одном чеке)
- Поля `quantity`, `price`, `summ` могут быть равны 0 для некоторых товаров
- Единицы измерения могут быть `л` (литры), `шт` (штуки) и другие
- Дата используется для заполнения тега 1178 (дата документа-основания) при коррекции

## Специфические особенности форматов

### Разделители и кодировка
- Все CSV-файлы используют точку с запятой (`;`) как разделитель полей
- Файлы сохраняются в кодировке UTF-8 с BOM
- Заголовки столбцов находятся в первой строке файла

### Типы данных
- Числовые значения могут содержать десятичную точку
- Текстовые поля могут содержать пробелы и специальные символы
- Все поля обязательны, но могут содержать нулевые значения (0 для чисел, пустая строка для текста)

### Валидация данных
- Суммы должны быть положительными числами
- Тип оплаты должен быть 0 или 1
- Фискальные признаки должны содержать только цифры
- Даты должны соответствовать формату `YYYY-MM-DD HH:MM:SS`

## Использование в проекте

### В файле `correction_final.py`
- `list.csv` читается для получения основных данных о чеках
- `receipts_data.csv` читается для получения данных о товарах (опционально)

### В файле `prepare_data.py`
- Создает `list.csv` из Excel-файла
- Извлекает фискальные признаки из гиперссылок
- Определяет тип оплаты по данным о наличных и безналичных платежах

### В файле `parse_ofd_receipts.py`
- Создает `receipts_data.csv` путем парсинга чеков с Платформы ОФД
- Извлекает данные о товарах, количестве, ценах и датах

## Ошибки и исключения

### Возможные ошибки в формате
- Неправильный разделитель (запятая вместо точки с запятой)
- Неправильная кодировка
- Отсутствие заголовков
- Несоответствие количества полей
- Неправильный формат чисел или дат

### Обработка ошибок
- Скрипты проверяют формат файлов при чтении
- При ошибках формата выводится сообщение об ошибке
- Некоторые ошибки могут привести к пропуску некорректных строк
