# Описание файлов проекта коррекции чеков ККТ

## Основные исполняемые файлы

### `correction_final.py`
Главный скрипт для проведения чеков коррекции на ККТ. Управляет всем процессом коррекции чеков, включая подключение к ККТ, обработку данных и пробивание чеков коррекции.

**Основные функции:**
- Подключение к ККТ через COM-интерфейс
- Чтение данных из CSV файлов
- Исключение чеков с НДС 10%
- Пробивание двух чеков коррекции для каждого чека: "Возврат прихода" и "Приход" с НДС 22%
- Логирование всех операций
- Сохранение состояния обработки

### `prepare_data.py`
Скрипт для подготовки данных из Excel в CSV формат. Преобразует данные из Excel-файла в формат, подходящий для обработки основным скриптом.

**Основные функции:**
- Чтение Excel файла с чеками
- Извлечение фискальных признаков из гиперссылок
- Определение типа оплаты (наличные/безналичные)
- Фильтрация чеков с НДС 10%
- Создание CSV файла с подготовленными данными

### `parse_ofd_receipts.py`
Скрипт для парсинга онлайн-чеков с Платформы ОФД. Извлекает данные о товарах из чеков по ссылке.

**Основные функции:**
- Чтение Excel файла с гиперссылками на чеки
- Извлечение параметров чека из URL
- HTTP-запросы к Платформе ОФД
- Парсинг HTML для извлечения информации о товарах
- Сохранение данных о товарах в CSV формате

## Входные и выходные файлы данных

### `Список чеков без НДС Финал.xlsx`
Исходный Excel-файл со списком чеков для коррекции. Содержит информацию о чеках, включая гиперссылки на Платформу ОФД.

### `list.csv`
Выходной CSV-файл с подготовленными данными чеков. Содержит:
- Сумму чека
- Тип оплаты (0 - безналичные, 1 - наличные)
- Фискальный признак (ФП)

### `receipts_data.csv`
Выходной CSV-файл с детальными данными о товарах из чеков. Содержит:
- Номер строки в исходном Excel
- Фискальный признак
- Наименование товара
- Количество
- Единица измерения
- Цена
- Сумма
- Дата чека

### `vat10_fp.txt`
Файл со списком фискальных признаков чеков с НДС 10%. Эти чеки исключаются из обработки.

### `processed.json`
Файл с перечнем уже обработанных фискальных признаков. Используется для возобновления прерванной работы.

### `correction_process.log`
Файл лога операций коррекции. Содержит все действия, выполненные скриптом, с временными метками.
