# Проект коррекции чеков ККТ

Проект для автоматизации проведения чеков коррекции на ККТ Штрих-М через драйвер ККТЛаб v.5.20.

## Назначение

Проект предназначен для массовой коррекции чеков, которые были пробиты без НДС, но должны были содержать НДС 22% (новая ставка, действующая с 01.01.2026).

### Алгоритм работы

Для каждого чека выполняется два чека коррекции:

1. **Чек коррекции "Возврат прихода"** — отмена ошибочного чека без НДС
2. **Чек коррекции "Приход"** — правильный чек с НДС 22%

> **Важно:** Сумма чека НЕ изменяется. НДС 22% выделяется из суммы по формуле: `НДС = Сумма × 22 / 122`

## Требования

### Программные требования

- Windows (для работы с COM-объектами драйвера ККТ)
- Python 3.8+
- Драйвер ККТ ККТЛаб v.5.20

### Python-зависимости

```bash
pip install pandas openpyxl requests pywin32
```

## Использование

### Шаг 1: Подготовка данных из Excel

```bash
python prepare_data.py
```

Скрипт читает `Список чеков без НДС Финал.xlsx` и создаёт `list.csv` с данными чеков:
- Сумма чека
- Тип оплаты (0 — безналичные, 1 — наличные)
- Фискальный признак (ФП)

### Шаг 2: Загрузка данных о товарах (опционально)

```bash
python parse_ofd_receipts.py
```

Скрипт загружает данные о товарах из чеков через API Платформы ОФД и сохраняет в `receipts_data.csv`.

### Шаг 3: Проведение чеков коррекции

```bash
python correction_final.py
```

## Настройки

### Основные параметры в `correction_final.py`

```python
# Режим работы: 'test' - тестовый (1 чек), 'prod' - полный
MODE = 'test'  # Изменить на 'prod' для реальной работы

# Настройки подключения к ККТ
COM_PORT = 3
BAUD_RATE = 115200

# НДС 22% (действует с 01.01.2026)
VAT_RATE = 22
```

### Исключение чеков с НДС 10%

Чеки с НДС 10% автоматически исключаются из обработки. Список ФП таких чеков задаётся в переменной `VAT_10_FP`:

```python
VAT_10_FP = [
    '607805230', '3560220730', '572166526', ...
]
```

## Формат данных

### list.csv

Основной файл с данными чеков для коррекции:

```csv
summ;type;fiscal_sign
0.37;0;2703755211
1799.55;0;2444685719
1000.00;1;2361559314
```

| Поле | Описание | Пример |
|------|----------|--------|
| `summ` | Сумма чека в рублях | `1799.55` |
| `type` | Тип оплаты: 0 = карта (безнал), 1 = наличные | `0` |
| `fiscal_sign` | Фискальный признак чека (ФП) | `2444685719` |

### receipts_data.csv

Файл с детальными данными о товарах из чеков (загружается с Платформы ОФД):

```csv
row;fp;name;quantity;unit;price;summ;date
2;2703755211;ДТ-А-К5;0.004;л;92.5;0.37;2026-01-01 00:06:00
3;2444685719;АИ-92-К5;27.0;л;66.65;1799.55;2026-01-01 00:06:00
10;1852047038;АИ-95-К5;45.0;л;66.7;3001.5;2026-01-01 00:16:00
10;1852047038;Капучино (Время Кофе) с корицей 300мл;0;л;0;220.0;2026-01-01 00:16:00
```

| Поле | Описание | Пример |
|------|----------|--------|
| `row` | Номер строки в исходном Excel | `2` |
| `fp` | Фискальный признак чека | `2703755211` |
| `name` | Наименование товара | `АИ-92-К5` |
| `quantity` | Количество (в литрах или штуках) | `27.0` |
| `unit` | Единица измерения: `л` = литры, `шт` = штуки | `л` |
| `price` | Цена за единицу | `66.65` |
| `summ` | Сумма позиции | `1799.55` |
| `date` | Дата и время чека | `2026-01-01 00:06:00` |

## Особенности работы

### Возобновление прерванной работы

Скрипт сохраняет список обработанных чеков в `processed.json`. При повторном запуске уже обработанные чеки пропускаются.

### Тестовый режим

По умолчанию скрипт работает в тестовом режиме (`MODE = 'test'`), обрабатывая только один чек. Для полной обработки измените режим на `'prod'`.

## Логирование

Все действия логируются в файл `correction_process.log` с временными метками:

```

```

## Технические детали

### Работа с ККТ

Скрипт использует COM-объект `AddIn.DrvFR` для взаимодействия с драйвером ККТ:

- `FNOpenCheckCorrection()` — открытие чека коррекции (ФФД 1.2)
- `FNOperation()` — добавление позиции в чек
- `FNCloseCheckEx()` — закрытие чека
- `FNSendTag()` — отправка TLV-тегов (дата основания, ФП корректируемого чека)

### TLV-теги

Используемые теги в чеках коррекции:

| Тег | Описание |
|-----|----------|
| 1178 | Дата документа-основания |
| 1192 | ФП корректируемого чека |
