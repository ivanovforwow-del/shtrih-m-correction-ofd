# Рекомендации по использованию проекта коррекции чеков ККТ

## Обзор

Данный документ содержит лучшие практики и рекомендации по эффективному и безопасному использованию проекта коррекции чеков ККТ.

## Подготовительные мероприятия

### 1. Планирование процесса коррекции

#### Оценка объема работ
- Проведите анализ Excel-файла перед началом обработки
- Оцените количество чеков, требующих коррекции
- Определите примерное время, необходимое для обработки

#### Проверка качества данных
- Убедитесь, что Excel-файл содержит все необходимые колонки
- Проверьте, что фискальные признаки корректны и содержат только цифры
- Убедитесь, что суммы чеков положительны и разумны

#### Проверка настройки НДС
- Подтвердите, что ставка НДС (`VAT_RATE`) установлена правильно
- Проверьте список чеков с НДС 10% (`VAT_10_FP`) на актуальность
- Убедитесь, что в проекте учитываются все необходимые ставки НДС

### 2. Подготовка инфраструктуры

#### Подготовка ККТ
- Убедитесь, что ККТ исправна и готова к работе
- Проверьте уровень бумаги в ККТ
- Убедитесь, что ККТ зарегистрирована с нужными реквизитами
- Подготовьте тестовую ККТ для проверки перед использованием боевой системы

#### Подготовка данных
- Создайте резервные копии всех исходных файлов
- Убедитесь, что Excel-файл не открывается в других приложениях
- Подготовьте чистую рабочую директорию для проекта

## Безопасность и надежность

### 1. Безопасность операций

#### Использование тестового режима
- Всегда начинайте с тестового режима (`MODE = 'test'`)
- Проверьте, что коррекция одного чека проходит успешно
- Убедитесь, что результаты коррекции соответствуют ожиданиям
- Только после успешного тестирования переключайтесь в прод режим

#### Проверка на тестовой ККТ
- Если возможно, используйте отдельную ККТ для тестирования
- Проверьте, что все операции коррекции проходят корректно
- Убедитесь, что фискальные документы формируются правильно

#### Контроль идентификаторов
- Проверьте, что фискальные признаки уникальны и корректны
- Убедитесь, что не происходит дублирования операций
- Используйте файл `processed.json` для отслеживания выполнения

### 2. Надежность процесса

#### Обработка ошибок
- Мониторьте файл `correction_process.log` в реальном времени
- Убедитесь, что система корректно обрабатывает ошибки
- Проверьте, что при сбоях система может восстановиться

#### Восстановление после прерываний
- Проверьте, что файл `processed.json` корректно обновляется
- Убедитесь, что система может возобновить работу с места прерывания
- Тестируйте сценарий принудительного завершения процесса

## Эффективность и производительность

### 1. Оптимизация процесса

#### Параллельная подготовка данных
- Запустите `prepare_data.py` заранее для подготовки `list.csv`
- Используйте `parse_ofd_receipts.py` для предварительного получения данных о товарах
- Оптимизируйте структуру Excel-файла для более быстрой обработки

#### Мониторинг прогресса
- Используйте информацию в логе для отслеживания прогресса
- Убедитесь, что обработка идет с ожидаемой скоростью
- При необходимости корректируйте параметры для оптимизации

#### Управление ресурсами
- Мониторьте использование памяти при обработке больших файлов
- Убедитесь, что система не испытывает нехватки ресурсов
- При необходимости оптимизируйте обработку больших объемов данных

### 2. Качество данных

#### Валидация данных
- Проверяйте корректность фискальных признаков перед обработкой
- Убедитесь, что суммы чеков и типы оплаты корректны
- Используйте валидацию данных в `prepare_data.py` для проверки

#### Обработка особых случаев
- Обратите внимание на чеки со смешанной оплатой
- Проверьте, что чеки с НДС 10% корректно исключаются из обработки
- Убедитесь, что чеки с некорректными данными помечаются для ручной обработки

## Рекомендации по настройке

### 1. Оптимальные параметры

#### Настройка подключения к ККТ
- Убедитесь, что `COM_PORT` установлен правильно
- Обычно `BAUD_RATE = 115200` подходит для большинства ККТ
- Проверьте стабильность соединения перед началом массовой обработки

#### Настройка параметров обработки
- Используйте `REQUEST_DELAY` в `parse_ofd_receipts.py` для предотвращения блокировки
- Убедитесь, что `MODE` установлен в правильное значение перед запуском
- Проверьте, что `VAT_RATE` соответствует текущему законодательству

### 2. Настройка безопасности

#### Ограничение доступа
- Ограничьте доступ к файлам проекта только авторизованным лицам
- Защитите доступ к ККТ от несанкционированного использования
- Используйте права доступа к файлам для предотвращения случайных изменений

#### Резервное копирование
- Создавайте резервные копии файлов данных перед запуском
- Регулярно сохраняйте копии файлов `processed.json` и `correction_process.log`
- Обеспечьте возможность восстановления после сбоев

## Мониторинг и обслуживание

### 1. Постоянный мониторинг

#### Мониторинг логов
- Регулярно проверяйте `correction_process.log` на наличие ошибок
- Убедитесь, что все операции коррекции проходят успешно
- Мониторьте производительность процесса

#### Мониторинг состояния
- Проверяйте содержимое `processed.json` для отслеживания прогресса
- Убедитесь, что количество обработанных чеков увеличивается
- Контролируйте соответствие обработанных чеков ожидаемому объему

### 2. Регулярное обслуживание

#### Обновление списков
- Регулярно обновляйте список чеков с НДС 10% (`VAT_10_FP`)
- Проверяйте актуальность Excel-файлов перед каждой обработкой
- Обновляйте параметры НДС при изменениях в законодательстве

#### Проверка работоспособности
- Периодически тестируйте подключение к ККТ
- Проверяйте корректность форматов данных
- Убеждайтесь в актуальности версий зависимостей

## Рекомендации по масштабированию

### 1. Обработка больших объемов

#### Пакетная обработка
- Разделяйте большие объемы данных на части для обработки
- Используйте файл `processed.json` для отслеживания выполнения
- Планируйте обработку на ночное время для минимизации влияния на бизнес-процессы

#### Мониторинг ресурсов
- Мониторьте использование памяти и процессора
- При необходимости оптимизируйте обработку больших файлов
- Учитывайте влияние на общую систему при планировании

### 2. Автоматизация

#### Планирование задач
- Используйте планировщик задач Windows для автоматизации подготовки данных
- Настройте уведомления о завершении процессов
- Планируйте регулярные проверки состояния системы

#### Мониторинг производительности
- Ведите статистику по времени обработки
- Анализируйте узкие места в процессе
- Планируйте оптимизацию на основе собранной статистики

## Заключение

Следование этим рекомендациям поможет обеспечить безопасное, надежное и эффективное использование проекта коррекции чеков ККТ. Всегда начинайте с тестового режима, создавайте резервные копии и внимательно мониторьте процесс выполнения. При возникновении проблем обращайтесь к документации и файлам логов для диагностики.
