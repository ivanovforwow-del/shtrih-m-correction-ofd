# Установка и запуск проекта коррекции чеков ККТ

## Системные требования

### Операционная система
- Windows 10 или выше (необходима для работы COM-объектов драйвера ККТ)

### Программное обеспечение
- Python 3.8 или выше
- Драйвер ККТ ККТЛаб v.5.20
- Microsoft Excel (для работы с исходными файлами)

## Установка зависимостей

### Установка Python

Если Python еще не установлен:

1. Загрузите установщик Python с [официального сайта](https://www.python.org/downloads/)
2. Установите Python 3.8 или выше
3. Убедитесь, что Python добавлен в PATH во время установки

### Установка Python-пакетов

Откройте командную строку и выполните:

```bash
pip install pandas openpyxl requests pywin32
```

### Установка драйвера ККТ

1. Установите драйвер ККТ ККТЛаб v.5.20
2. Подключите ККТ к компьютеру
3. Убедитесь, что драйвер корректно установлен и распознает ККТ

## Подготовка проекта

### Клонирование или скачивание проекта

Скопируйте файлы проекта в выбранную директорию.

### Подготовка исходных данных

1. Подготовьте Excel-файл `Список чеков без НДС Финал.xlsx` с чеками для коррекции
2. Убедитесь, что файл содержит необходимые колонки:
   - Дата/время
   - Наличными
   - Электронными
   - Посмотреть чек (с гиперссылками на Платформу ОФД)
   - НДС 10% (если используется для фильтрации)

## Настройка проекта

### Настройка параметров подключения к ККТ

Откройте файл `correction_final.py` и проверьте параметры:

```python
# Настройки подключения к ККТ
COM_PORT = 3          # Порт подключения (обычно COM3, COM4 и т.д.)
BAUD_RATE = 115200    # Скорость передачи данных
```

Определите правильный COM-порт в диспетчере устройств Windows.

### Настройка режима работы

В том же файле установите режим работы:

```python
# Режим работы: 'test' - тестовый (1 чек), 'prod' - полный
MODE = 'test'  # Для начала используйте 'test', после проверки измените на 'prod'
```

### Обновление списка чеков с НДС 10%

Если у вас есть чеки с НДС 10%, которые не требуют коррекции, обновите массив `VAT_10_FP` в `correction_final.py`:

```python
VAT_10_FP = [
    '607805230', '3560220730', '572166526',  # и другие ФП
]
```

## Подготовка данных

### Запуск подготовки данных

Для преобразования Excel-файла в формат, подходящий для обработки, выполните:

```bash
python prepare_data.py
```

Скрипт создаст файл `list.csv` с основными данными чеков:
- Сумма чека
- Тип оплаты (0 — безналичные, 1 — наличные)
- Фискальный признак (ФП)

### (Опционально) Загрузка данных о товарах

Если вы хотите использовать детализацию товаров из чеков, выполните:

```bash
python parse_ofd_receipts.py
```

Скрипт извлечет данные о товарах из чеков через API Платформы ОФД и сохранит в `receipts_data.csv`.

## Запуск коррекции чеков

### Проверка конфигурации

Перед запуском полной обработки убедитесь в правильности настроек:

1. Проверьте подключение к ККТ
2. Убедитесь, что в режиме `test` скрипт корректно работает с одним чеком
3. Проверьте, что файлы `list.csv` и `receipts_data.csv` существуют и содержат корректные данные

### Запуск в тестовом режиме

Для проверки работы выполните:

```bash
python correction_final.py
```

В тестовом режиме будет обработан только один чек.

### Запуск в полном режиме

После успешной проверки измените режим на `'prod'` в файле `correction_final.py`:

```python
MODE = 'prod'
```

Затем снова запустите:

```bash
python correction_final.py
```

## Мониторинг и логирование

### Проверка логов

Все операции логируются в файл `correction_process.log`. В случае проблем проверьте этот файл.

### Проверка прогресса

Скрипт показывает прогресс в реальном времени, включая:
- Количество обработанных чеков
- Количество ошибок
- Процент выполнения

### Восстановление после прерываний

Если процесс был прерван, при повторном запуске скрипт пропустит уже обработанные чеки, так как их фискальные признаки сохраняются в `processed.json`.

## Возможные проблемы и решения

### Проблемы с подключением к ККТ

- Убедитесь, что драйвер ККТ правильно установлен
- Проверьте правильность COM-порта
- Убедитесь, что ККТ включена и готова к работе
- Проверьте права доступа к порту

### Проблемы с Excel-файлами

- Убедитесь, что формат Excel совместим с pandas/openpyxl
- Проверьте, что гиперссылки в файле имеют правильный формат
- Убедитесь, что имена колонок совпадают с ожидаемыми

### Проблемы с данными

- Проверьте, что суммы в чеках положительные
- Убедитесь, что типы оплаты указаны правильно (0 или 1)
- Проверьте, что фискальные признаки содержат только цифры

## Рекомендации по безопасности

1. Всегда сначала запускайте в тестовом режиме
2. Делайте резервные копии важных файлов перед запуском
3. Мониторьте процесс выполнения
4. Проверяйте результаты коррекции в личном кабинете налоговой службы
