# Решение проблем и часто задаваемые вопросы

## Часто задаваемые вопросы

### 1. Что делать, если скрипт не может подключиться к ККТ?

**Проблема:** При запуске скрипта возникает ошибка подключения к ККТ.

**Решение:**
1. Проверьте, включена ли ККТ и находится ли она в готовом состоянии
2. Убедитесь, что ККТ подключена к компьютеру через USB или другой интерфейс
3. Проверьте правильность настройки COM-порта в файле `correction_final.py`
4. Убедитесь, что драйвер ККТ ККТЛаб v.5.20 установлен корректно
5. Проверьте, не используется ли ККТ другим приложением
6. Убедитесь, что у пользователя есть права доступа к COM-порту

### 2. Почему некоторые чеки пропускаются при обработке?

**Проблема:** В логе видно, что часть чеков пропускается с сообщением "уже обработан".

**Решение:** Это нормальное поведение системы. При повторном запуске скрипт пропускает уже обработанные чеки, фискальные признаки которых сохранены в файле `processed.json`. Чтобы обработать все чеки заново:
1. Удалите или очистите файл `processed.json`
2. Перезапустите скрипт

### 3. Как обрабатываются чеки со смешанной оплатой?

**Проблема:** В логе появляются сообщения о необходимости ручной обработки некоторых чеков.

**Решение:** Чеки, содержащие одновременно наличные и безналичные платежи, требуют специальной обработки, так как чек коррекции должен быть однотипным. Такие чеки помечаются для ручной обработки и пропускаются автоматической системой.

### 4. Почему в чеках коррекции НДС выделяется из суммы?

**Проблема:** Непонятно, почему сумма чека не меняется, но НДС рассчитывается как процент от суммы.

**Решение:** Это происходит потому, что в новых правилах НДС 22% должен быть выделен из суммы чека, а не добавлен к ней. Формула расчета: `НДС = Сумма × 22 / 122`. Это соответствует ситуации, когда сумма чека включает в себя НДС.

## Частые ошибки и их решения

### Ошибка: "win32com not available"

**Проблема:** При запуске появляется предупреждение о недоступности win32com.

**Решение:**
```bash
pip install pywin32
```

Если проблема сохраняется после установки:
```bash
python Scripts/pywin32_postinstall.py -install
```

### Ошибка: "ModuleNotFoundError: No module named 'pandas'"

**Проблема:** Один из необходимых модулей не найден.

**Решение:**
```bash
pip install pandas openpyxl requests pywin32
```

### Ошибка: "PermissionError: [Errno 13] Permission denied"

**Проблема:** Ошибка доступа к файлу или порту.

**Решение:**
1. Убедитесь, что файл не открыт в другом приложении
2. Запустите командную строку от имени администратора
3. Проверьте права доступа к файлам проекта

### Ошибка: "Access is denied" при работе с COM-объектом

**Проблема:** Ошибка доступа к драйверу ККТ.

**Решение:**
1. Запустите Python/скрипт от имени администратора
2. Проверьте, не запущено ли другое приложение, использующее ККТ
3. Перезапустите драйвер ККТ

## Проблемы с Excel-файлами

### Проблема: "Error reading Excel: [Errno 2] No such file"

**Решение:**
1. Убедитесь, что файл `Список чеков без НДС Финал.xlsx` находится в той же директории, что и скрипт
2. Проверьте правильность имени файла
3. Убедитесь, что файл не открыт в Excel

### Проблема: "Column 'Посмотреть чек' not found"

**Решение:**
1. Проверьте, что в Excel-файле есть столбец с названием "Посмотреть чек"
2. Убедитесь, что регистр букв соответствует ожидаемому
3. Проверьте, что структура Excel-файла соответствует ожидаемой

### Проблема: Не извлекаются фискальные признаки из гиперссылок

**Решение:**
1. Проверьте, что гиперссылки в Excel содержат параметр `fp=` с фискальным признаком
2. Убедитесь, что формат гиперссылок соответствует ожидаемому: `...&fp=1234567890...`

## Проблемы с обработкой данных

### Проблема: Все чеки с НДС 10% обрабатываются как обычные

**Решение:**
1. Проверьте, что фискальные признаки чеков с НДС 10% добавлены в массив `VAT_10_FP` в `correction_final.py`
2. Убедитесь, что формат ФП в списке совпадает с тем, что в данных (только цифры)

### Проблема: Неправильно определяется тип товара (обычный/подакцизный)

**Решение:**
1. Проверьте, что названия товаров содержат ключевые слова для определения топлива
2. Убедитесь, что массив `FUEL_KEYWORDS` в `correction_final.py` содержит все необходимые ключевые слова
3. При необходимости добавьте новые ключевые слова в массив

## Проблемы с парсером чеков

### Проблема: Парсер не может извлечь данные о товарах из чеков

**Решение:**
1. Проверьте, что URL-адреса чеков на Платформе ОФД действительны
2. Убедитесь, что интернет-соединение стабильно
3. Проверьте, не изменился ли формат HTML-страницы чеков на платформе
4. Увеличьте значение `REQUEST_DELAY` в `parse_ofd_receipts.py`, если получаете ошибки связанные с частыми запросами

### Проблема: В результате парсинга получаются пустые данные

**Решение:**
1. Проверьте, что гиперссылки в Excel содержат корректные параметры (id, date, fp)
2. Убедитесь, что чеки на Платформе ОФД доступны и содержат информацию о товарах
3. Проверьте формат данных в чеках - возможно, структура HTML изменилась

## Проблемы с производительностью

### Проблема: Обработка занимает слишком много времени

**Решение:**
1. Убедитесь, что `REQUEST_DELAY` в `parse_ofd_receipts.py` не установлен в слишком большое значение
2. Проверьте стабильность интернет-соединения (для парсера)
3. Убедитесь, что ККТ работает корректно и не вызывает задержек

### Проблема: Большое количество ошибок при обработке

**Решение:**
1. Проверьте качество исходных данных в Excel-файле
2. Убедитесь, что все обязательные поля заполнены корректно
3. Проверьте, не превышаете ли вы лимиты API Платформы ОФД (для парсера)

## Отладка и диагностика

### Включить детальный лог

Для получения более подробной информации о процессе работы можно добавить дополнительные сообщения в лог. В файле `correction_final.py` можно расширить функцию `log()` или добавить дополнительные сообщения в ключевых местах.

### Проверка подключения к ККТ

Прежде чем запускать полную обработку, протестируйте подключение к ККТ отдельно:

```python
from correction_final import connect_kkt
kkt = connect_kkt()
if kkt:
    print("Подключение успешно")
    # можно выполнить простую операцию для проверки
else:
    print("Ошибка подключения")
```

### Тестирование в изолированном режиме

Для отладки конкретной функции можно создать тестовый скрипт, который вызывает только эту функцию с подготовленными тестовыми данными.
