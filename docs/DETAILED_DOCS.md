# Подробная документация проекта коррекции чеков ККТ

## Обзор

Проект предназначен для автоматизации проведения чеков коррекции на ККТ Штрих-М через драйвер ККТЛаб v.5.20. Он позволяет массово корректировать чеки, которые были пробиты без НДС, но должны были содержать НДС 22%.

## Архитектура проекта

### 1. `correction_final.py` - основной модуль коррекции

Это главный файл, который управляет процессом коррекции чеков. Он подключается к ККТ через COM-интерфейс и проводит чеки коррекции.

#### Основные функции:

- `correction_refund()` - создает чек коррекции "Возврат прихода" (отмена ошибочного чека)
- `correction_sale()` - создает чек коррекции "Приход" с правильным НДС
- `process_corrections()` - основной цикл обработки чеков
- `connect_kkt()` - подключение к ККТ
- `load_items_data()` - загрузка данных о товарах из CSV

#### Технические особенности:

- Использует COM-объект `AddIn.DrvFR` для взаимодействия с драйвером ККТ
- Работает с тегами ФФД 1.2 (теги 1178 и 1192)
- Автоматически определяет тип товара (обычный/подакцизный) и единицы измерения
- Поддерживает обработку чеков с товарами и без них
- Имеет встроенную систему логирования и отслеживания обработанных чеков

#### Работа с тегами:

- Тег 1178: Дата документа-основания (формат: ДД.ММ.ГГГГ)
- Тег 1192: ФП корректируемого чека
- Тег 1212: Признак предмета расчета (1 - товар, 2 - подакцизный товар)

### 2. `prepare_data.py` - подготовка данных

Скрипт для преобразования данных из Excel-файла в формат, подходящий для обработки.

#### Основные функции:

- Чтение Excel файла с помощью pandas и openpyxl
- Извлечение фискального признака из гиперссылок
- Определение типа оплаты (наличные/безналичные)
- Фильтрация чеков с НДС 10%
- Обработка случаев смешанной оплаты

#### Особенности:

- Поддерживает различные форматы гиперссылок в Excel
- Обнаруживает и пропускает чеки с НДС 10%
- Обрабатывает ошибки при парсинге данных
- Создает CSV файл с полями: сумма, тип оплаты, фискальный признак

### 3. `parse_ofd_receipts.py` - парсер чеков

Скрипт для извлечения данных о товарах из чеков на Платформе ОФД.

#### Основные функции:

- Чтение Excel файла с гиперссылками на чеки
- Извлечение параметров чека из URL (id, date, fp)
- HTTP-запросы к Платформе ОФД для получения HTML чека
- Парсинг HTML для извлечения информации о товарах
- Сохранение данных в CSV формате

#### Особенности:

- Использует библиотеку requests для HTTP-запросов
- Имеет задержку между запросами для предотвращения блокировки
- Обрабатывает HTML-сущности в названиях товаров
- Извлекает данные о количестве, единицах измерения и ценах товаров

## Работа с ККТ

### Подключение

Скрипт использует COM-объект `AddIn.DrvFR` для взаимодействия с драйвером ККТ ККТЛаб v.5.20. Подключение осуществляется через последовательный порт (COM) с заданными параметрами скорости передачи данных.

### Типы чеков коррекции

1. **Чек коррекции "Возврат прихода"**:
   - Использует CheckType = 2
   - Отменяет ошибочный чек без НДС
   - Товары добавляются без НДС
   - Отправляются теги 1178 и 1192

2. **Чек коррекции "Приход"**:
   - Использует CheckType = 0
   - Пробивает правильный чек с НДС 22%
   - Сумма остается неизменной, НДС выделяется из суммы
   - Также отправляются теги 1178 и 1192

### Расчет НДС

НДС 22% выделяется из суммы по формуле: `НДС = Сумма × 22 / 122`. Это соответствует ситуации, когда сумма включает НДС.

### Определение типа товара

Скрипт автоматически определяет тип товара по ключевым словам в названии:

- Подакцизные товары: топливо (АИ-92, АИ-95, ДТ-, бензин, дизель и т.д.)
- Признак предмета расчета: 2 для подакцизных товаров, 1 для обычных товаров
- Единицы измерения: литры для топлива, штуки для других товаров

## Обработка ошибок и восстановление

### Система логирования

Все операции записываются в файл `correction_process.log` с временной меткой. Это позволяет отследить ход выполнения и выявить возможные проблемы.

### Восстановление после прерываний

Состояние обработки сохраняется в файле `processed.json`, который содержит список уже обработанных фискальных признаков. При повторном запуске скрипт пропускает уже обработанные чеки.

### Обработка исключений

Каждая операция с ККТ оборачена в блок try-catch для предотвращения аварийного завершения программы. При ошибках скрипт может отменить чек и продолжить обработку следующих чеков.

## Тестовый режим

Скрипт имеет два режима работы:

- `test`: обрабатывает только один чек, удобен для проверки
- `prod`: полная обработка всех чеков

## Файловая структура данных

### Основные файлы

- `list.csv`: основной список чеков для коррекции
- `receipts_data.csv`: детализация товаров по чекам
- `processed.json`: список уже обработанных чеков
- `vat10_fp.txt`: список фискальных признаков с НДС 10%
- `correction_process.log`: лог операций

### Форматы файлов

#### list.csv
```
summ;type;fiscal_sign
1799.55;0;2444685719
```

#### receipts_data.csv
```
row;fp;name;quantity;unit;price;summ;date
3;2444685719;АИ-92-К5;27.0;л;66.65;1799.55;2026-01-01 00:06:00
```

## Установка и настройка

### Требования

- Windows OS (для работы COM-объектов)
- Python 3.8+
- Драйвер ККТ ККТЛаб v.5.20
- Установленные пакеты: pandas, openpyxl, requests, pywin32

### Настройка

1. Убедитесь, что драйвер ККТ правильно установлен и настроен
2. Проверьте параметры подключения в `correction_final.py` (COM_PORT, BAUD_RATE)
3. Подготовьте Excel-файл с чеками для коррекции
4. Запустите `prepare_data.py` для создания `list.csv`
5. При необходимости запустите `parse_ofd_receipts.py` для получения детализации товаров
6. Запустите `correction_final.py` для выполнения коррекции

## Безопасность и надежность

- Скрипт включает проверки для предотвращения дублирования операций
- Используется пошаговая обработка с возможностью восстановления
- Все операции логируются для аудита
- Система обнаруживает и пропускает чеки с НДС 10% для предотвращения ошибок
