# Настройка проекта коррекции чеков ККТ

## Обзор

Данный документ описывает все настройки и конфигурационные параметры проекта, необходимые для его корректной работы.

## correction_final.py

### Основные настройки

#### `CSV_FILE`
- **Тип:** строка
- **Значение по умолчанию:** `'list.csv'`
- **Описание:** Имя основного CSV-файла с данными чеков для коррекции

#### `ITEMS_FILE`
- **Тип:** строка
- **Значение по умолчанию:** `'receipts_data.csv'`
- **Описание:** Имя CSV-файла с данными о товарах из чеков

#### `LOG_FILE`
- **Тип:** строка
- **Значение по умолчанию:** `'correction_process.log'`
- **Описание:** Имя файла лога операций

#### `PROCESSED_FILE`
- **Тип:** строка
- **Значение по умолчанию:** `'processed.json'`
- **Описание:** Имя файла с перечнем уже обработанных фискальных признаков

### Режим работы

#### `MODE`
- **Тип:** строка
- **Значение по умолчанию:** `'prod'`
- **Возможные значения:** `'test'`, `'prod'`
- **Описание:** Режим работы скрипта
  - `'test'` - тестовый режим, обрабатывается только один чек
  - `'prod'` - производственный режим, обрабатываются все чеки

### Настройки подключения к ККТ

#### `COM_PORT`
- **Тип:** целое число
- **Значение по умолчанию:** `3`
- **Описание:** Номер COM-порта, к которому подключена ККТ

#### `BAUD_RATE`
- **Тип:** целое число
- **Значение по умолчанию:** `115200`
- **Описание:** Скорость передачи данных по COM-порту

### Настройки НДС

#### `VAT_RATE`
- **Тип:** целое число
- **Значение по умолчанию:** `22`
- **Описание:** Ставка НДС, которая будет применяться в чеках коррекции

### Типы товаров и единицы измерения

#### `SUBJECT_TYPE_GOOD`
- **Тип:** целое число
- **Значение по умолчанию:** `1`
- **Описание:** Признак обычного товара (тег 1212)

#### `SUBJECT_TYPE_EXCISE`
- **Тип:** целое число
- **Значение по умолчанию:** `2`
- **Описание:** Признак подакцизного товара (тег 1212)

#### `MEASURE_UNIT_PIECE`
- **Тип:** целое число
- **Значение по умолчанию:** `0`
- **Описание:** Единица измерения "штуки"

#### `MEASURE_UNIT_LITER`
- **Тип:** целое число
- **Значение по умолчанию:** `41`
- **Описание:** Единица измерения "литр"

#### `MEASURE_UNIT_GRAM`
- **Тип:** целое число
- **Значение по умолчанию:** `10`
- **Описание:** Единица измерения "грамм"

#### `MEASURE_UNIT_KILOGRAM`
- **Тип:** целое число
- **Значение по умолчанию:** `11`
- **Описание:** Единица измерения "килограмм"

### Ключевые слова для определения топлива

#### `FUEL_KEYWORDS`
- **Тип:** массив строк
- **Значение по умолчанию:**
  ```
  ['АИ-92', 'АИ-95', 'АИ-98', 'АИ-80', 'ДТ-', 'ДТ ', 'дизель', 'бензин', 'К5', 'газомоторн']
  ```
- **Описание:** Ключевые слова, по которым определяется, что товар является топливом

### Исключение чеков с НДС 10%

#### `VAT_10_FP`
- **Тип:** массив строк
- **Значение по умолчанию:**
  ```
  [
      '607805230', '3560220730', '572166526', '760931914', '4075689541',
      '1035906310', '896116170', '1827453878', '2163166593', '1066150790',
      '2713257049', '1825692796', '2808322941'
  ]
  ```
- **Описание:** Список фискальных признаков чеков с НДС 10%, которые будут исключены из обработки

## Требования к окружению

### Программные зависимости

Для работы проекта необходимо установить следующие Python-пакеты:
```
pandas
openpyxl
requests
pywin32
```

### Системные требования

- Операционная система: Windows (необходима для работы COM-объектов драйвера ККТ)
- Версия Python: 3.8 или выше
- Драйвер ККТ: ККТЛаб v.5.20

## Рекомендации по настройке

### Настройка COM-порта

1. Определите, к какому COM-порту подключена ваша ККТ
2. Проверьте настройки порта в диспетчере устройств Windows
3. Установите соответствующее значение в переменной `COM_PORT`

### Настройка скорости передачи данных

- Обычно значение по умолчанию `BAUD_RATE = 115200` подходит для большинства ККТ
- В редких случаях может потребоваться изменение на 9600, 19200 или 57600

### Режим работы

- Для тестирования всегда используйте `MODE = 'test'`
- После проверки работоспособности измените на `MODE = 'prod'` для полной обработки

### Обновление списка чеков с НДС 10%

- Регулярно обновляйте массив `VAT_10_FP` с новыми фискальными признаками
- Это предотвратит случайную обработку чеков, которые не требуют коррекции
