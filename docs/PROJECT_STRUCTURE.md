# Структура проекта коррекции чеков ККТ

## Обзор архитектуры

Проект состоит из нескольких модулей, каждый из которых выполняет свою специфическую функцию в процессе коррекции чеков:

- `correction_final.py` - основной модуль, управляющий процессом коррекции чеков
- `prepare_data.py` - модуль подготовки данных из Excel в формат, подходящий для обработки
- `parse_ofd_receipts.py` - модуль извлечения данных о товарах из чеков на Платформе ОФД
- `README.md` - основная документация проекта
- `docs/` - директория с дополнительной документацией

## Файловая структура

```

├── prepare_data.py              # Подготовка данных из Excel

```

## Основные компоненты

### correction_final.py

Это сердце проекта, которое управляет всем процессом коррекции чеков:

- Подключается к ККТ через COM-интерфейс
- Читает данные из `list.csv` и `receipts_data.csv`
- Исключает чеки с НДС 10% на основе списка в `vat10_fp.txt`
- Для каждого чека:
  - Пробивает чек коррекции "Возврат прихода" (отмена ошибочного чека)
  - Пробивает чек коррекции "Приход" (с правильным НДС 22%)
- Сохраняет состояние обработки в `processed.json`
- Логирует все операции в `correction_process.log`

### prepare_data.py

Модуль преобразования данных из Excel в формат, подходящий для обработки:

- Читает Excel файл `Список чеков без НДС Финал.xlsx`
- Извлекает фискальные признаки из гиперссылок
- Определяет тип оплаты (наличные/безналичные)
- Фильтрует чеки с НДС 10%
- Создает CSV файл `list.csv` с полями: сумма, тип оплаты, фискальный признак

### parse_ofd_receipts.py

Модуль извлечения детализации товаров из чеков:

- Читает Excel файл и извлекает гиперссылки на чеки
- Извлекает параметры чека (id, date, fp) из URL
- Выполняет HTTP-запросы к Платформе ОФД
- Парсит HTML-ответ для извлечения информации о товарах
- Создает CSV файл `receipts_data.csv` с информацией о товарах

## Интеграция компонентов

Процесс работы проекта включает несколько этапов:

1. **Подготовка данных** - запуск `prepare_data.py` для преобразования Excel в CSV
2. **(Опционально) Извлечение данных о товарах** - запуск `parse_ofd_receipts.py`
3. **Коррекция чеков** - запуск `correction_final.py` для пробивания чеков коррекции

Каждый этап зависит от результатов предыдущего, хотя второй этап является опциональным.

## Безопасность и надежность

Проект включает механизмы для обеспечения безопасности и надежности:

- Проверки для предотвращения дублирования операций
- Пошаговая обработка с возможностью восстановления
- Система логирования для аудита
- Обнаружение и пропуск чеков с НДС 10%
- Защита от обработки чеков со смешанной оплатой

## Расширяемость

Архитектура проекта позволяет легко добавлять новые функции:

- Модульная структура позволяет расширять функциональность каждого компонента
- API четко определены и документированы
- Система логирования помогает отслеживать работу новых функций
- Механизм обработки состояния позволяет возобновлять работу после добавления новых чеков
